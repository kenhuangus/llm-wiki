## Claims
- Insecure Default File Permissions in Local Filesystem Memory Tool (Confidence: 0.98, Source: [[d5c84097]])
- The local filesystem memory tool in the Anthropic Python SDK created memory files with mode 0o666, leaving them world-readable on systems with a standard umask and world-writable in environments with a permissive umask such as many Docker base images. (Confidence: 0.95, Source: [[d5c84097]])
- A local attacker on a shared host could read persisted agent state, and in containerized deployments could modify memory files to influence subsequent model behavior. (Confidence: 0.92, Source: [[d5c84097]])
- Both the synchronous and asynchronous memory tool implementations were affected. (Confidence: 0.88, Source: [[d5c84097]])
- Users on the affected versions are advised to update to the latest version. (Confidence: 0.97, Source: [[d5c84097]])

## Claims
- Threat actors are using HTTP cookies as a control channel for PHP-based web shells on Linux servers to achieve remote code execution. (Confidence: 1.0, Source: [[9506509c]])
- PHP web shells rely on cookie values to gate execution instead of exposing command execution through URL parameters or request bodies. (Confidence: 1.0, Source: [[9506509c]])
- PHP web shells persist via Cron on Linux servers. (Confidence: 1.0, Source: [[9506509c]])
